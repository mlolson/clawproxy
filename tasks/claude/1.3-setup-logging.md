# Task 1.3: Set Up Logging

**Assignee:** Claude
**Dependencies:** 1.2
**Status:** Open

## Description

Configure tracing-based logging for both binaries with environment-based filtering.

## Acceptance Criteria

- [ ] Add tracing initialization function
- [ ] Configure `EnvFilter` for RUST_LOG support
- [ ] Initialize tracing in both binaries
- [ ] Verify: `RUST_LOG=debug cargo run --bin clawproxy` shows debug output

## Implementation

```rust
// Can be in lib.rs or each binary

use tracing_subscriber::{fmt, prelude::*, EnvFilter};

pub fn init_tracing() {
    tracing_subscriber::registry()
        .with(fmt::layer())
        .with(EnvFilter::from_default_env())
        .init();
}
```

## Usage in Binaries

```rust
// src/bin/clawproxy.rs
fn main() {
    clawproxy::init_tracing();
    // or inline initialization

    tracing::info!("Starting clawproxy...");
}
```

## Log Levels

- `error` - Unrecoverable errors
- `warn` - Recoverable issues, deprecations
- `info` - High-level operations (server start, request handling)
- `debug` - Detailed operation info
- `trace` - Very verbose, request/response bodies

## Notes

- Default to `info` level when RUST_LOG is not set
- Consider adding structured fields for request IDs later
